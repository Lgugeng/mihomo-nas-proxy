# 安全策略

## 报告安全漏洞

如果你发现了安全漏洞，请**不要**通过公开 Issue 报告。

请通过以下方式联系我们：
- 发送邮件至：[你的邮箱]
- 或通过 GitHub 的 "Report a vulnerability" 功能

我们会在 48 小时内回复。

## 安全最佳实践

### 部署时

1. **修改默认密码**：`secret` 必须改为强密码
2. **限制网络访问**：API 默认监听 `127.0.0.1`，不要随意改为 `0.0.0.0`
3. **使用防火墙**：只开放必要端口（7890/7891/9090）
4. **定期更新**：关注 Mihomo 和 MetaCubeXD 的安全更新

### 配置时

1. **不要泄露订阅链接**：订阅链接等同于账号密码
2. **不要提交敏感信息到 Git**：使用 `.env` 文件管理
3. **定期轮换密钥**：定期更换 `secret`

### 运维时

1. **查看日志**：定期检查 `docker logs mihomo` 是否有异常
2. **备份配置**：使用 `make backup` 定期备份
3. **监控资源**：注意 CPU 和内存使用情况

## 已知安全限制

- `network_mode: host` 模式下容器共享主机网络栈，无法通过 Docker 网络隔离
- `allow-lan: true` 时局域网内其他设备可使用代理，请确保局域网可信
